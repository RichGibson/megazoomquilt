# Thumbnail Generation

## How thumbnails are generated

### Local panos (no `tile_base_url`)

The Flask `/thumbnail/<pano_id>` route generates thumbnails on first request:

1. Reads `{pano_id}.json` for image dimensions (`width`, `height`, `levels`)
2. Picks the lowest zoom level where:
   - The content is at least `THUMB_MIN_DIM` (128px) on the shorter side
   - The total tile count does not exceed `THUMB_MAX_TILES` (16)
3. Composites all tiles in the grid at that zoom level into one PIL image
4. Crops to the actual content dimensions (`content_w × content_h`), removing
   quadtree padding tiles beyond the image boundary
5. Saves to `static/panos/{id}/{id}_thumb.jpg` (cached for future requests)

### R2 panos (`tile_base_url` set)

Thumbnails are pre-generated by `util/generate_r2_thumbs.py` (same algorithm as
above but fetching tiles from R2 via HTTP) and stored on R2 at:

```
{tile_base_url}/{pano_id}_thumb.jpg
```

The `index.html` template links directly to this URL. The Flask thumbnail route
is not involved.

## Known issues

### 1. Black padding at top of content (primary issue)

Gigapan quadtree tiling pads images to power-of-2 tile counts. For some images
the actual content occupies the bottom portion of the tile grid, with black-filled
tiles above. The current crop `(0, 0, content_w, content_h)` includes this
padding, resulting in thumbnails that are mostly black with content at the bottom.

**Fix wanted:** show full image height, center-crop width if the image aspect ratio
is wider than the target thumbnail aspect ratio. Never crop top or bottom.

### 2. Non-rectangular images

Some panoramas have non-rectangular content (irregular edges, curved stitching
artifacts) surrounded by black fill. The thumbnail shows the bounding rectangle
including the black surround, which can be misleading.

**TODO:** investigate smarter cropping for non-rectangular images — e.g. detect
the content bounding box and crop to that before compositing.

### 3. Zoom level selection

The current heuristic (first zoom level with ≤16 tiles and ≥128px short side) may
not always produce the best visual result. For very wide panoramas the chosen zoom
level can be quite coarse.

## Iterating on thumbnails

Use `util/gen_thumb.py` to generate a thumbnail for a single pano without
running the full app:

```bash
python3 util/gen_thumb.py 1000259
python3 util/gen_thumb.py 1000259 --zoom 0
python3 util/gen_thumb.py 1000259 --out /tmp/test_thumb.jpg
```

This overwrites the cached thumbnail by default so the app picks it up immediately.
